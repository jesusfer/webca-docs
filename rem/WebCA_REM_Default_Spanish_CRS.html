    3.3.3.1 Registro de usuarios

    Casos de uso relacionados con el registro y administración de los usuarios que van a solicitar certificados digitales de la CA.
    UC-0001 Registro de un usuario anónimo
    Versión 1.0 ( 16/02/2018 )
    Autores
    • Jesús Fernández
    Fuentes
    • Jesús Fernández
    Dependencias
  • [OBJ-0009] Llevar un registro de los usuarios del sistema
  • [IRQ-0001] Usuario
  • [IRQ-0003] Solicitud Certificado
    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un usuario anónimo accede a la aplicación web
    Precondición El usuario debe tener una cuenta de correo electrónico pública
    Secuencia normal Paso Acción
    1 El actor Usuario anónimo (ACT-0004) accede al sistema iniciando una nueva sesión en el sistema
    2 El sistema da al usuario tres opciones: autenticarse, registrarse o recuperar una cuenta
    3 El actor Usuario anónimo (ACT-0004) elige la opción de registrar un nuevo usuario
    4 El sistema solicita al nuevo usuario los datos necesarios para registrar un nuevo usuario
    5 El actor Usuario anónimo (ACT-0004) rellena los datos solicitados por el sistema
    6 El sistema procesa el registro del nuevo usuario y lo pone en la cola de solicitudes de certificado de usuario
    7 Si el sistema está configurado para permitir el registro automático de nuevos usuarios, el sistema autoriza la emisión del certificado de usuario
    8 Si el sistema está configurado para que el registro de nuevos usuarios esté supervisado por un operador, el actor Operador (ACT-0002) revisa la petición de registro de nuevo usuario y, si acepta el registro, autoriza la emisión del certificado de usuario
    9 El sistema procesa la autorización del registro de nuevo usuario, expide el certificado de usuario y envia el certificado al usuario
    Postcondición El usuario recibe un certificado de usuario que le permite autenticarse en la aplicación web
    Excepciones Paso Acción
    8 Si el Operador no acepta el registro del nuevo usuario, el sistema comunica al usuario el motivo del rechazo, a continuación este caso de uso queda sin efecto
    Rendimiento Paso Tiempo máximo

    Frecuencia esperada PD
    Importancia vital
    Urgencia inmediatamente

    UC-0002 Renovación o reemisión de un certificado de usuario
    Versión 1.0 ( 16/02/2018 )
    Autores
    • Jesús Fernández
    Fuentes
    • Jesús Fernández
    Dependencias
  • [OBJ-0009] Llevar un registro de los usuarios del sistema
    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando un usuario registrado quiere acceder al sistema, pero su certificado de usuario ha expirado o se ha perdido
    Precondición El usuario debe estar registrado en el sistema
    Secuencia normal Paso Acción
    1 El actor Usuario registrado (ACT-0003) accede al sistema iniciando una nueva sesión
    2 El sistema da al usuario tres opciones: autenticarse, registrarse o recuperar una cuenta
    3 El actor Usuario registrado (ACT-0003) elige la opción de recuperar una cuenta de usuario
    4 El sistema solicita la dirección de correo electrónico con la que el usuario está registrado en el sistema y envía un código único con el que el usuario puede confirmar que es el propietario de esa dirección de correo electrónico
    5 El actor Usuario registrado (ACT-0003) recibe el correo electrónico e inserta el código único en el sistema
    6 El sistema recibe el código único, confirma que el código corresponde con una solicitud de renovación, expide un nuevo certificado de usuario y se lo envía al usuario
    Postcondición El usuario recibe un certificado de usuario nuevo
    Excepciones Paso Acción
    6 Si el código único no corresponde con una solicitud de renovacion vigente, el sistema comunica el fallo al usuario, a continuación este caso de uso queda sin efecto
    Rendimiento Paso Tiempo máximo
    - -
    Frecuencia esperada PD
    Importancia vital
    Urgencia hay presión
    Estado en construcción
    Estabilidad PD

    • Descripción El sistema deberá comportarse tal como se describe en el siguiente caso de uso cuando el usuario piensa que se ha comprometido el certificado de usuario que usa para autenticarse en el sistema
    Precondición El usuario debe estar registrado en el sistema
    Secuencia normal Paso Acción

    5 Si el usuario confirma la acción, el sistema revoca el certificado del usuario
    Postcondición El certificado de usuario queda revocado
    Excepciones Paso Acción
    5 Si el usuario no confirma la acción, el sistema dejará sin efecto la revocación del certificado, a continuación este caso de uso queda sin efecto
